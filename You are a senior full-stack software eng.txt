--- app/timetable/algorithm.py ---
Implement the FULL adaptive timetable generation algorithm:

Class TimetableGenerator:
  - __init__(self, user_id, study_hours_per_day=8, break_duration=15)
  - Method generate(tasks) → returns list of TimetableSlot objects
  
  Algorithm logic:
  1. Sort tasks by deadline (urgent first), then by difficulty (harder first)
  2. Calculate weight for each task: weight = (difficulty * 2) + urgency_score
     urgency_score = max(0, 10 - days_until_deadline)
  3. Distribute study slots across 7 days proportionally by weight
  4. Insert 15-minute breaks every 90 minutes of study
  5. Avoid scheduling past 10 PM or before 7 AM
  6. Return balanced slot list with day, start_time, end_time, subject, task_id
  
  Include helper methods:
  - _calculate_urgency(deadline) → float
  - _allocate_hours(tasks, total_available_hours) → dict {task_id: hours}
  - _create_slots(allocations) → list of TimetableSlot

--- app/timetable/routes.py ---

- POST /timetable/generate → run TimetableGenerator for current user's pending tasks, save to DB
- GET /timetable/api/slots → JSON API for calendar rendering
- POST /timetable/clear → delete all slots for current user
